📖 第四章 · 必须理解的 5 个核心概念

下面的概念决定了你用得顺不顺手，花 5 分钟读懂：

1️⃣ 工作目录（Working Directory）

Codex 只能看到和操作当前工作目录里的文件。换项目时记得在顶部切换目录——AI 看不到目录以外的东西。

2️⃣ 上下文（Context）

Codex 会记住当前对话里的所有内容——你说过的话、它创建的文件、报过的错误。对话太长后可能会"忘掉"最早的内容，如果发现 AI 开始犯糊涂，用 /clear 清空上下文重新开始。

3️⃣ 权限模式（Permissions）

三种模式，在设置里切换：Auto（推荐）项目目录内自动执行，目录外需确认；Read-only 只看不写，适合阅读陌生代码；Full Access 完全自主，适合你完全信任 AI 的场景。

4️⃣ 模型选择（Model）

在设置里可以切换模型。GPT-5 最强但需要 Plus 订阅，GPT-4o 也够用。如果你用的是 API Key，选贵的模型效果好但烧钱快。

5️⃣ Token（用量）

Token = AI 处理文本的计量单位。大约 1 个中文字 = 2 个 Token，1000 个英文单词 = 1500 个 Token。每次对话消耗 Token，ChatGPT Plus 订阅用户有额度上限，API Key 用户按量扣费。

📝 第七章 · CODEX.md 项目配置

在项目根目录创建一个 CODEX.md 文件，Codex 每次打开这个项目都会自动读取。它相当于给 AI 提前规定好工作规范：

# CODEX.md — 这个项目的 AI 工作规范

## 项目信息
- 这是一个个人博客项目，React + TypeScript + Tailwind CSS
- 后端用 Supabase，数据库是 PostgreSQL
- 部署在 Vercel 上

## 代码规范（AI 必须遵守）
- 组件用函数式写法，不要用 Class 组件
- 所有组件 props 必须定义 TypeScript interface
- 状态管理用 Zustand，不要引入 Redux
- 错误必须 try-catch 处理，禁止吞异常
- 不要使用 any 类型

## 禁止事项
- 不要修改 package.json 中的依赖版本
- 不要修改 .env 或 .env.local 文件
- 不要删除任何已有文件，除非我明确要求
- 不要引入超过 100KB 的新 npm 包

## 我的偏好
- 用中文回复和注释
- 写代码之前先解释方案，等我确认再动手
- 每改完一个文件，自动运行相关测试
✅
实战建议：每一个重要项目都应该有一个 CODEX.md。建好之后，AI 就不会乱写代码风格、乱装依赖——相当于你提前给 AI 上了一道紧箍咒。花 5 分钟写一个，后续省你 100 次纠正 AI 的口水。

🔌 第八章 · MCP 扩展能力

MCP（Model Context Protocol）是给 Codex 增加额外能力的扩展接口。桌面版内置了基础能力，进阶用户可以手动添加更多。通俗理解：MCP 就像给 Codex 装插件——装了浏览器插件就能操作浏览器，装了数据库插件就能查数据库。

🌐 Playwright MCP

操作浏览器：打开网页、点击、填表单、截图。让 AI 能"看到"你的网页长什么样，前端调试必备。

必装MCP

📂 Filesystem MCP

增强文件操作：搜索文件内容、批量重命名、跨目录操作。桌面版内置了基础版本，这个更强大。

推荐MCP

🗄️ SQLite MCP

直接操作数据库：建表、查数据、分析统计。后端项目或数据分析场景的神器。

推荐MCP

🐙 GitHub MCP

管理 GitHub 仓库：创建 Issue、查看 PR、发布 Release。团队协作效率翻倍。

可选MCP
💡
新手建议：桌面版开箱即用，不需要折腾 MCP。等你用了 1-2 周熟悉基本操作后，再根据需要添加。MCP 配置在设置 → MCP 中添加，格式参考上方配置建议章节。

📝 CODEX.md — 项目级 AI 指令

在项目根目录创建 CODEX.md，Codex 会自动读取并遵守其中的规范。以下是一个经过实战打磨的模板：

# 项目编码规范

## 技术栈
- 前端：React 18 + TypeScript + Tailwind CSS
- 后端：Python FastAPI + PostgreSQL

## 代码规范
- 组件用函数式 + Hooks
- 所有函数必须有类型注解
- 错误必须显式处理
- 文件命名：组件 PascalCase，工具 camelCase

## 禁止
- 不要生成包含 API 密钥的代码
- 不要在没有确认的情况下删除文件

📋 Codex 配置文件

Codex 使用 ~/.codex/config.toml（TOML 格式），而不是 JSON：

# Codex 配置文件 ~/.codex/config.toml

[model]
default = "gpt-5"           # 默认模型

[permissions]
mode = "auto"               # auto | read-only | full-access

[approval]
enabled = true              # 敏感操作需要确认

[mcp]
# MCP 服务器配置（见下方插件推荐）

[env]
OPENAI_API_KEY = "sk-你的密钥"
✅
三种权限模式：
· Auto（推荐）：项目目录内自由操作，目录外需确认
· Read-only：只能读代码和回答问题，不能改文件
· Full Access：完全自主，包括网络访问

📋 MCP 服务器配置

Codex 同样支持 MCP 扩展。在 ~/.codex/config.toml 中添加：

[mcp.servers.playwright]
command = "npx"
args = ["@playwright/mcp", "--browser", "chrome"]

[mcp.servers.filesystem]
command = "npx"
args = ["-y", "@modelcontextprotocol/server-filesystem", "/Users/你的用户名"]
